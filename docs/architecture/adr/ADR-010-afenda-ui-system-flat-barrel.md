# 1. Purpose

Record the decision to retire the `@afenda/ui` (`packages/design-system`) playground-gateway design system and replace it with a greenfield `@afenda/ui-system` (`packages/ui-system`) whose only public surface is one **flat barrel** plus one stylesheet. The goal is a solid, simple, consistency-first UI package optimized for AI-assisted coding: minimal third-party influence, stable owned tokens, and a single obvious import door — no gateway indirection and no `*Contract` type layer.

This ADR supersedes [ADR-009](ADR-009-afenda-ui-playground-gateway.md).

---

# 2. Scope

## 2.1 In Scope

- Greenfield `@afenda/ui-system` package replacing `@afenda/ui` as the sole UI design-system SSOT
- The single flat public barrel (`@afenda/ui-system`) and the single stylesheet subpath (`@afenda/ui-system/styles.css`)
- Radix primitive base pinned via the shadcn `new-york` style (unified `radix-ui` package), lucide icons, and **no external/paid component registries**
- CSS ownership split: the consuming app owns Tailwind + the animation layer + `@source`; the package owns only semantic tokens
- Source-package model (no build step) consumed via Next.js `transpilePackages`
- Guardrail tests and the `apps/web` barrel-only boundary
- Retirement/deletion of `packages/design-system` and the old UI-pipeline skills

## 2.2 Out of Scope

- Product feature UIs built on top of the package (each is its own change)
- Any return of a Next.js `/playground` route harness (remains absent; requires an explicit Shadcn Studio MCP slice — unchanged from ADR-009 § D5)
- Multi-tenancy / RBAC ([ARCH-023](../ARCH-023-multi-tenancy.md))

# 3. Decision

## Context

ADR-009 consolidated two colliding packages into `@afenda/ui` and trimmed its exports to a `./playground` gateway, adding a `*Contract extends` type per gateway component. In practice this was heavier than the product needs: a gateway indirection, a curated contract layer, a large unpublished template surface (`views/`, `fake-db/`, …) kept on disk, and a dependency on the paid Shadcn Studio registries. The user's explicit direction: deprecate that system and build a new package that is **solid, simple, and consistency-first**, with minimal external noise and an import surface that AI agents cannot drift from.

## Decision (binding)

### D1 — Greenfield package, single SSOT

`packages/ui-system` (`@afenda/ui-system`) is the sole UI design-system package. `packages/design-system` (`@afenda/ui`) is retired and deleted; the playground gateway and `*Contract` pattern are removed with it.

### D2 — One flat barrel, one stylesheet (the entire public API)

```json
"exports": {
  ".": { "types": "./src/index.ts", "default": "./src/index.ts" },
  "./styles.css": "./src/styles/tokens.css"
}
```

`src/index.ts` re-exports every primitive under `src/components/ui/**` plus `cn`. Consumers import only `@afenda/ui-system` (components) and `@afenda/ui-system/styles.css` (tokens). There is no gateway subpath, no `*Contract` type, and no deep-import surface. The barrel carries no `"use client"` — each interactive primitive keeps its own directive, preserved across re-export.

### D3 — Radix pinned, owned source, no registry noise

`components.json` pins the shadcn `new-york` style, which emits the unified `radix-ui` package (`import { Dialog as DialogPrimitive } from "radix-ui"`), with `iconLibrary: lucide` and **no `registries` key**. Components are owned source generated by the shadcn CLI and are freely regenerable; on generation their `@/` aliases are converted to relative imports so resolution is bulletproof for `tsc`, Next, and Vitest without a `paths`/`imports` map. `radix-ui` is catalog-locked in `pnpm-workspace.yaml`.

### D4 — CSS ownership: app owns Tailwind, package owns tokens

The consuming app (`apps/web/globals.css`) owns `@import "tailwindcss"`, the animation layer (`tw-animate-css`), the `@source` registration of the package, and brand typography. The package's `styles.css` owns **only** the semantic token layer (`@theme inline`, `:root`, `.dark`, `@custom-variant dark`) — it never imports Tailwind. This keeps one Tailwind compilation owner and lets the package stay a pure token + component source.

### D5 — Source package, no build step

`@afenda/ui-system` is consumed directly from TypeScript source via Next.js `transpilePackages` (and `optimizePackageImports`). No `dist`, no separate build/semver pipeline.

### D6 — Guardrails over convention

Committed tests enforce the contract: barrel coverage (every `components/ui/*` primitive is re-exported), single public surface, no external registries, lucide-only icons, no raw color literals, and keyboard/focus smoke for Dialog/Sheet/DropdownMenu/Select/Tooltip. `apps/web/__tests__/ui-boundary.test.ts` asserts the barrel + stylesheet are the only allowed specifiers and bans any deep import or retired `@afenda/ui` reference. A Tailwind compile smoke confirms package classes are emitted via `@source`. Canonical shadcn upstream patterns in vendored `components/ui/**` are kept green by a scoped Biome override (documented for 1:1 upstream parity).

## Alternatives considered

| Option | Verdict |
|--------|---------|
| **A** Keep ADR-009 `@afenda/ui` gateway + `*Contract` | Rejected — heavier than needed; gateway indirection and contract layer add drift surface and paid-registry coupling |
| **B** Base UI primitive base (shadcn default `base-nova`) | Rejected — Radix pinned for maturity/stability; `--base radix` / `new-york` style |
| **C** Built package (dist + d.ts) | Rejected — source package is simpler for a monorepo-internal consumer; no publish pipeline needed |
| **D** Keep external Shadcn Studio registries | Rejected — the user's explicit goal is minimal third-party influence; free-tier shadcn + Radix only |

## Consequences

| | |
|--|--|
| **+** | One obvious import door (AI cannot drift); owned stable tokens; no paid-registry coupling; regenerable canonical source; guardrails fail on any drift |
| **−** | Future `shadcn add` emits alias imports that must be converted to relative (documented one-step workflow); vendored `components/ui/**` carries a scoped Biome override |
| **=** | Full `next build` RSC verification depends on a present `.env.local`; barrel/typecheck/interaction tests and a Tailwind compile smoke cover the pipeline in the interim |
