## Refined Command — Cinematic Ice ⇄ Fire Lynx Authentication Scene

Use the two supplied lynx artworks as **two states of one continuous cinematic environment**, not as separate slideshow backgrounds.

Create a full-screen authentication experience using **React/Next.js, Tailwind CSS v4, and CSS animations**. The scene must feel intelligent, atmospheric, premium, and physically connected to the login panel.

## Stronger creative direction

The key improvement is to treat the login panel as the **source and receiver of the transformation**:

- In the Ice state, the panel contains frozen energy.
- The frame melts and releases heat.
- The heat travels through the keyhole into the Lynx.
- The Lynx ignites.
- During the Fire state, incoming snow strikes the panel.
- Frost begins on the panel and spreads back through the keyhole.
- The Lynx freezes again.

This creates a meaningful visual loop instead of a decorative background exchange.

### Recommended cycle

|   Time | Scene    | Dominant event                   |
| -----: | -------- | -------------------------------- |
|   0–4s | Ice      | Frozen stillness                 |
|   4–7s | Thaw     | Panel frost melts                |
|  7–10s | Ignition | Fire travels from panel to Lynx  |
| 10–15s | Fire     | Full obsidian/fire state         |
| 15–18s | Snow     | Snow and embers coexist          |
| 18–22s | Freeze   | Frost spreads from panel to Lynx |
| 22–25s | Ice      | Environment stabilizes           |

The most important implementation instruction is:

> **Never transition the entire image with one opacity animation. Use regional masks and a narrow moving transformation boundary so the Lynx appears to change material rather than change pictures.**
