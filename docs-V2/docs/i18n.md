## Upstream ladder (reference only)

Do **not** paste these into Lite without a named Docs i18n reopen + Scratch update.

### 1. Config — `lib/i18n.ts`

```ts
// Upstream example — NOT Lite
import { defineI18n } from "fumadocs-core/i18n";

export const i18n = defineI18n({
  defaultLanguage: "en",
  languages: ["en", "cn"],
});
```

Options: upstream [i18n config](https://fumadocs.dev/docs/headless/internationalization/config) (`hidePrefix`, etc.).

### 2. Middleware — `proxy.ts`

```ts
// Upstream example — NOT Lite (docs app only — never apps/web/proxy.ts)
import { createI18nMiddleware } from "fumadocs-core/i18n/middleware";
import { i18n } from "@/lib/i18n";

export default createI18nMiddleware(i18n);

export const config = {
  matcher: ["/((?!api|_next/static|_next/image|favicon.ico).*)"],
};
```

Upstream: default middleware is optional; custom / third-party middleware must align with `hidePrefix`. Lite: no docs locale middleware until reopen.

### 3. Translations — `layout.shared` + `RootProvider`

```tsx
// Upstream example — NOT Lite
export const translations = i18n
  .translations()
  .extend(uiTranslations())
  .add({
    en: { displayName: "English" },
    cn: { displayName: "Chinese" },
  });

export function baseOptions(locale: string): BaseLayoutProps {
  return { /* locale-specific nav / links */ };
}

// app/[lang]/layout.tsx
<RootProvider i18n={i18nProvider(translations, lang)}>{children}</RootProvider>
```

UI packs: upstream [Translations](https://fumadocs.dev/docs/ui/translations).

### 4. Routing — `app/[lang]/…`

Move pages/layouts under `app/[lang]/`; keep route handlers (e.g. `app/api/search`) **outside** the lang segment. After the move, keep `global.css` import paths valid.

### 5. Pass locale

| Surface | Upstream change |
|---------|-----------------|
| `lib/source.ts` | `loader({ i18n, … })` |
| Docs layout | `tree={source.getPageTree(lang)}` · `baseOptions(lang)` |
| Docs page | `source.getPage(slug, lang)` · `getPages(lang)` |
| `generateStaticParams` | If segment ≠ `lang`: `source.generateParams('slug', 'locale')` |

### 6. Search

Configure Orama (or cloud) for locales — upstream [Orama i18n](https://fumadocs.dev/docs/headless/search/orama#internationalization). Lite search stays English fetch mode until reopen.

### 7. Writing documents · navigation

- Content layout: [page-conventions i18n routing](https://fumadocs.dev/docs/page-conventions#i18n-routing) (`dot` / `dir` parsers).
- Non-Fumadocs links: prepend locale via `useParams().lang` or `DynamicLink` (`href="/[lang]/…"`).

## When reopen is allowed

Explicit Docs i18n reopen must cover:

1. Approved locale list (start small — do **not** import Xerp multi-locale packs)
2. `lib/i18n.ts` + docs-only `proxy.ts` (`createI18nMiddleware`) — **not** `apps/web/proxy.ts`
3. Move to `app/[lang]/**` (layouts/pages); keep `app/api/**` outside; fix CSS imports
4. `loader({ i18n })` · `getPageTree(lang)` · `getPage(slug, lang)` · `baseOptions(locale)` · `i18nProvider`
5. Content split (`dir` or `dot`) + `meta.json` per locale — [page-conventions.md](page-conventions.md)
6. Orama / search locale matrix — update [search-orama.md](search-orama.md)
7. OpenAPI: **default Lite choice** = keep operation MDX English-only; localize narrative MDX first
8. If AsyncAPI also reopens: extend with `asyncapiTranslations` — [asyncapi.md](asyncapi.md) (`createAsyncAPIPage` chapter)
9. Wire tests + Scratch update here (flip Status to Active when shipped)
10. Docs project rules still hold — no Neon / product secrets — [README.md](README.md)

Until then: English-only lock below.
