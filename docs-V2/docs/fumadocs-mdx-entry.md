# Fumadocs MDX — Accessing Collections (scratch)

| Field | Value |
|-------|-------|
| Surface | `docs-V2/docs/fumadocs-mdx-entry.md` |
| Authority | **Scratch** — upstream [Accessing Collections](https://fumadocs.dev/docs/mdx/entry) · [Server](https://fumadocs.dev/docs/mdx/entry/server) · [Browser](https://fumadocs.dev/docs/mdx/entry/browser) · [Dynamic](https://fumadocs.dev/docs/mdx/entry/dynamic) · disk `@afenda/docs` |
| Status | **Active** — `collections/server` only |
| Audience | Engineers importing collection outputs into the docs app |
| Updated | 2026-07-19 |

After `defineDocs` / `defineConfig`, Fumadocs MDX compiles collections and generates **entry files** under the output directory (Lite: `.source/`). Apps import via the `collections/*` path alias.

Getting Started: [fumadocs-mdx.md](fumadocs-mdx.md). Next wiring: [fumadocs-mdx-next.md](fumadocs-mdx-next.md). Loader: [loader-api.md](loader-api.md) · [loader-source.md](loader-source.md). Lazy/dynamic modes: [fumadocs-mdx-performance.md](fumadocs-mdx-performance.md).

## Lite lock (configured)

| Entry | Path | Lite |
|-------|------|------|
| Server | `collections/server` → `.source/server.ts` | **Shipped** — sole import |
| Browser | `collections/browser` → `.source/browser.ts` | **Outside baseline** — [fumadocs-mdx-browser.md](fumadocs-mdx-browser.md) |
| Dynamic | `collections/dynamic` → `.source/dynamic.ts` | **Outside baseline** — [fumadocs-mdx-dynamic.md](fumadocs-mdx-dynamic.md) |
| Alias | `tsconfig` `collections/*` → `./.source/*` | **Shipped** |
| Output dir | `.source/` (gitignored) | Generated by `generate:source` |
| Access pattern | `docs` → `loader({ source: docs.toFumadocsSource() })` | **Shipped** — not raw collection in pages |

```ts
// apps/docs/lib/source.ts
import { docs } from "collections/server";
import { loader } from "fumadocs-core/source";

export const source = loader({
  baseUrl: "/docs",
  source: docs.toFumadocsSource(),
  plugins: [openapi.loaderPlugin()],
});
```

```jsonc
// apps/docs/tsconfig.json
"paths": {
  "@/*": ["./*"],
  "collections/*": ["./.source/*"]
}
```

Do **not** import `collections/server` from client components — server entry only.

---

## Overview

Compilation is done by Fumadocs MDX bundler plugins (`createMDX`). Entry files are the typed surface for collection outputs. Upstream tree:

```text
.source/
  server.ts    # Lite — Active
  browser.ts   # Outside
  dynamic.ts   # Outside (needs docs.dynamic: true)
```

---

## Server Entry — Active

Upstream recommends using the collection with [`loader()`](https://fumadocs.dev/docs/headless/source-api) rather than reading the raw collection in every page.

| API | Lite |
|-----|------|
| `docs` from `collections/server` | Yes |
| `docs.toFumadocsSource()` | Yes |
| Separate `doc` collection + `toFumadocsSource(posts, [])` | Outside — no blog collection |
| `source.getPage` · `page.data.body` · `page.data.toc` | Yes — [get-toc.md](get-toc.md) |

```tsx
// apps/docs/app/docs/[[...slug]]/page.tsx (shape)
const page = source.getPage(params.slug);
const MDX = page.data.body;
return <MDX components={getMDXComponents({ … })} />;
```

## Outside baseline (summary)

| Pattern | Why |
|---------|-----|
| `collections/browser` · `createClientLoader` | No client collection entry — [fumadocs-mdx-browser.md](fumadocs-mdx-browser.md) |
| `collections/dynamic` · `page.data.load()` | Dynamic mode Outside — [fumadocs-mdx-dynamic.md](fumadocs-mdx-dynamic.md) |
| Direct `.source/server` deep import | Breaks alias contract |
| Second loader from a `doc`-only collection | Multi-source — [loader-source.md](loader-source.md) |
| Committing `.source/` | Gitignored — always regenerate |
